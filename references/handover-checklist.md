# Handover Checklist & Timeline

## T-30 Days: Start Planning

- [ ] 确认离职日期和最后工作日
- [ ] 指定接手人（或启动招聘）
- [ ] 安排交接访谈时间（至少 2 次，每次 1-2 小时）
- [ ] 整理项目清单和代码仓库权限

## T-21 Days: First Interview

- [ ] 运行 `handover_interview.py` 完成首轮访谈
- [ ] 生成 PROJECT.md 和 PEOPLE.md
- [ ] 确认所有代码已提交，分支已合并
- [ ] 文档链接整理（Wiki、Confluence、Notion 等）

## T-14 Days: Shadow Period

- [ ] 接手人开始参与日常开发/工作
- [ ] 离职人 Review 接手人的工作
- [ ] 完成 LESSONS.md 访谈
- [ ] 确认所有密码/密钥/权限已转移

## T-7 Days: Second Interview

- [ ] 完成 TODO.md 访谈
- [ ] 过一遍所有待办和已知问题
- [ ] 确认外部联系人（供应商、客户、合作伙伴）已介绍
- [ ] 安排离职后的紧急联系方式（可选）

## T-3 Days: Final Check

- [ ] 所有交接文档已完成并分享给团队
- [ ] 接手人能独立完成至少一个完整任务
- [ ] 权限转移确认（代码仓库、服务器、第三方服务）
- [ ] 团队群公告 / 告别安排

## T-0: Last Day

- [ ] 正式交接确认（双方签字或邮件确认）
- [ ] 权限回收（安全要求）
- [ ] 告别 🫡

## Post-Departure: Living Document

### Week 1
- [ ] 接手人遇到第一个独立问题，记录到 TODO.md
- [ ] 检查是否有遗漏的关键信息

### Month 1
- [ ] 回顾 LESSONS.md，补充新的发现
- [ ] 更新 PROJECT.md 中的过时信息
- [ ] 评估交接质量：1-10 分

### Month 3
- [ ] 正式评估交接是否完成
- [ ] 将个人经验提炼为团队 wiki 条目
- [ ] 反哺下一次的 handover 流程
