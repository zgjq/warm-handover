# Customized Interview Templates by Role

## Backend Engineer

### Technical Depth
- 数据库 schema 有哪些非直观的变更历史？为什么要改？
- API 接口的版本管理策略是什么？有哪些已废弃但还没下线的？
- 缓存策略：哪些数据可以缓存？哪些绝对不能？
- 定时任务 / cron：有哪些？各自的风险点？

### Operations
- 发布流程：从代码到上线的完整步骤
- 监控告警：哪些指标最重要？阈值是多少？
- 回滚方案：出问题了怎么快速回滚？

## Frontend Engineer

### Technical Depth
- 组件复用策略：哪些组件是团队共享的？
- 状态管理：为什么选这个方案？有哪些坑？
- 构建/部署：CI/CD 流程中前端相关的部分

### User Experience
- 有哪些用户反馈集中但还没解决的问题？
- 有哪些设计系统的约定是 "不说不知道" 的？

## Product Manager

### Knowledge
- 需求优先级是如何确定的？有哪些隐含规则？
- 有哪些 "老板说过但没写下来" 的方向性指示？
- 用户画像和核心指标：哪些数据最值得关注？

### Relationships
- 哪些 stakeholder 需要特别关注？各自的关注点？
- 有哪些跨部门的共识是 "大家都同意但没记录" 的？

## Designer

### Knowledge
- 设计系统的核心原则和禁区
- 有哪些 "试过不行" 的方案？为什么？
- 用户研究的发现有哪些没写进报告？

### Handoff
- 设计稿的标注规范
- 与前端协作的默契和常见摩擦点

## DevOps / SRE

### Infrastructure
- 架构图和关键路径
- 容量规划：当前瓶颈在哪里？
- 灾备方案：最坏情况怎么处理？

### Operations
- 日常巡检清单
- 供应商管理：合同到期日、关键联系人
- 安全合规：有哪些必须遵守的要求？
